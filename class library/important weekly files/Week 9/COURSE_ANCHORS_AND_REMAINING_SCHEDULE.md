# COURSE ANCHORS AND REMAINING SCHEDULE

## Current Phase

We are now in the **MVP Verification + Stabilization Phase**.

The goal is no longer just “build more.” The goal is to prove the project is real:

- it runs,
- the team understands it,
- the core flow is stable,
- bugs are visible,
- AI-assisted work is reviewed,
- progress is linked in GitHub.

## Sprint Structure

| Sprint | Weeks | Theme | Main question |
|---|---:|---|---|
| Sprint 3 | Weeks 9–11 | MVP Verification + Code Ownership | Does the MVP work, and can the team explain it? |
| Sprint 4 | Weeks 12–14 | QA + Refactor + Professional Polish | Is the app reliable enough to demo and hand off? |
| Final Sprint | Weeks 15–16 | Final Demo + Handoff | Can the team present, defend, and deliver the project professionally? |

## Weekly Progress Demo

Every class, each team must show one visible improvement.

5-minute format:

1. What changed since last week?
2. Show the improvement.
3. Show the evidence links.
4. One random member explains one code area, bug, test, or doc.
5. State one ask or blocker.

## Weekly Submission

Every week has one team submission:

- Weekly Sprint Packet Issue
- submitted by PM or Scribe
- due Friday after class by 23:59
- every student comments with 2–3 receipts

## New Rule

AI tools are allowed.

AI-assisted work only counts if the team can:

1. run it,
2. explain it,
3. test it,
4. debug it,
5. link evidence for it.

A feature is not done because AI generated it. A feature is done when the team can demo it, explain it, and prove it with GitHub evidence.

## What Students Should Stop Doing

- Saying “almost done” without evidence.
- Adding new features before the core flow is stable.
- Accepting AI code they cannot explain.
- Working without Issues.
- Opening giant PRs.
- Submitting screenshots without links.
- Waiting until Friday to remember GitHub exists.

## What Students Should Do Instead

- Pick one small issue.
- Create a branch.
- Use AI to inspect and plan before editing.
- Make a small change.
- Test it.
- Open a PR.
- Explain what changed.
- Link it in the Weekly Sprint Packet.
