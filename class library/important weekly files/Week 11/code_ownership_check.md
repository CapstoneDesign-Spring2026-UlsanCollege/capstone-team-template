# Week 11 Code Ownership Check

## Purpose

Every student must be able to explain one meaningful part of the project.

This can be code, docs, tests, bugs, setup, or an AI-assisted change.

## Good Ownership Areas

A student can own:

- one component,
- one function,
- one page or route,
- one API endpoint,
- one database table or model,
- one bug,
- one test,
- one setup step,
- one documentation page,
- one AI-assisted change they reviewed.

## Explanation Questions

Each student should be ready to answer:

1. What does this part do?
2. Where is it in the repo?
3. What user flow does it support?
4. What changed this week?
5. How did you test or verify it?
6. What could break here?
7. Did AI help with this? If yes, what did you check?

## What Good Looks Like

```md
Student: Mina
Owned area: Login form
File: src/components/LoginForm.jsx
Evidence: PR #22
Can explain: Clear
Summary: This form collects email and password, checks empty inputs, and sends the login request. I tested valid login, empty password, and wrong password.
```

## What Needs Work Looks Like

```md
Student: Joon
Owned area: Dashboard API
File: src/api/dashboard.js
Evidence: none yet
Can explain: Needs work
Problem: I can explain what it returns, but I do not understand the error handling yet.
Next step: Ask Copilot to explain the function, then write manual test steps.
```

## Reminder

Not understanding something yet is fixable.

Pretending to understand mystery code is not acceptable.
