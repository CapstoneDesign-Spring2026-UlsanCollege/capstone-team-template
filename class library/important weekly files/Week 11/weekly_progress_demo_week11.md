# Week 11 Weekly Progress Demo

## Purpose

Each team gets 5 minutes to show one visible improvement and prove progress with evidence.

No slides required unless they help.

Show the app. Show the repo. Show the evidence.

## Demo Roles

- **PM:** says what changed and states the ask/blocker
- **Demo Driver:** shows the working flow, fix, test, doc, or improvement
- **QA Lead / Scribe:** shows evidence links
- **Random member:** explains one code area, bug, test, doc, or AI-assisted change

## 5-Minute Format

| Segment | Time | Speaker | What to show |
|---|---:|---|---|
| What changed | 45 sec | PM | 1-3 specific changes |
| Demo | 2 min | Demo Driver | One visible improvement |
| Evidence check | 1 min | QA Lead/Scribe | PRs, Issues, tests, screenshots, docs |
| Ownership check | 1 min | Random member | Explain one part |
| Ask/blocker | 15 sec | PM | One specific need |

## What Counts as a Good Demo?

Good demos show one of these:

- working MVP core flow,
- fixed bug,
- improved error or empty state,
- test result,
- deployment/demo reliability improvement,
- useful docs update,
- AI-assisted code review result,
- refactor that keeps the app working.

## Weak Demo Examples

- “We worked on login.”
- “Almost done.”
- “AI generated it but we did not test it.”
- “The code is on someone’s laptop.”
- “We forgot to push.”

## Strong Demo Examples

- “We verified the signup flow. Here is Issue #24, PR #31, and the screenshot showing the success state.”
- “We found three P1 bugs in the dashboard flow. Here are the Issues and owners.”
- “We updated the setup guide and tested it on a teammate’s laptop.”
- “Copilot helped us inspect the create-post bug. We changed one file, tested manually, and linked the PR.”
