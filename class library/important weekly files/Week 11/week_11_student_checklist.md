# Week 11 Student Checklist

## Purpose

Use this checklist during class. Your team is proving that your MVP is real.

## Easy Win First

- [ ] Open your app.
- [ ] Open your repo.
- [ ] Open your GitHub Project board.
- [ ] Choose one core MVP flow to verify.
- [ ] Create or update one Issue: `Verify MVP core flow`.

## MVP Flow

Write your main user flow:

```md
Our main user can:
1.
2.
3.
4.
```

## Verification Checklist

- [ ] App runs locally or deployed version works.
- [ ] Demo path starts from a clear start state.
- [ ] User can complete the main action.
- [ ] User reaches a clear success state.
- [ ] At least one error or empty state is checked.
- [ ] Evidence is linked.
- [ ] Bug list is updated.
- [ ] At least one team member can explain the code path.

## Evidence Checklist

Add links to your Sprint Packet:

- [ ] Core flow Issue
- [ ] Pull Request
- [ ] Screenshot or video
- [ ] Test result or manual test notes
- [ ] Docs update, if needed
- [ ] Bug Issues, if something is broken

## Bug Checklist

For each bug, create or update an Issue.

- [ ] Bug has a clear title.
- [ ] Bug has severity: P0 / P1 / P2 / P3.
- [ ] Bug has an owner.
- [ ] Bug has evidence.
- [ ] Bug has a next step.

## AI Use Checklist

- [ ] We listed AI tools used this week.
- [ ] We explained what AI helped with.
- [ ] We explained what humans checked or changed.
- [ ] AI-assisted PRs include an AI Use Note.
- [ ] No one is claiming mystery code as done.

## Before You Leave Class

- [ ] Sprint Packet draft is started.
- [ ] Evidence links are added.
- [ ] Each student knows their receipts.
- [ ] Next week’s first task is written.
- [ ] Non-essential features are moved to `Nice Later`.

## Reminder

Almost done is not a status.

Show what works. Show what breaks. Link the evidence.
