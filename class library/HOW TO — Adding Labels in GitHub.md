# HOW TO — Adding Labels in GitHub

This guide shows you how to add labels to your repository.

Labels help your team organize work and quickly understand what type of task something is.

---

# What Labels Are Used For

In this class, labels are used to identify:

* bugs (something is broken)
* blockers (something is stopping progress)
* retros (team reflection)

Labels make Issues easier to understand at a glance.

---

# Step-by-Step Instructions

## Step 1 — Open Your Repository

Go to your team’s GitHub repository.

---

## Step 2 — Click “Issues”

At the top of the page, click:

**Issues**

---

## Step 3 — Click “Labels”

On the Issues page, click:

**Labels**

---

## Step 4 — Click “New label”

In the top right, click:

**New label**

---

## Step 5 — Create Labels

Create the following labels:

### Bug

Name:

```
bug
```

Description:

```
Something is broken
```

Color:
Red

---

### Blocker

Name:

```
blocker
```

Description:

```
Something is stopping progress
```

Color:
Orange

---

### Retro

Name:

```
retro
```

Description:

```
Sprint reflection
```

Color:
Blue or Purple

---

## Step 6 — Save Each Label

Click:

**Create label**

Repeat for each label.

---

# How to Use Labels

When creating an Issue:

1. Click “Labels”
2. Select the correct label

Examples:

* use **bug** when something is not working
* use **blocker** when you are stuck
* use **retro** when reflecting on a sprint

---

# Common Mistakes

Avoid these problems:

* not using labels at all
* using the wrong label
* creating too many unnecessary labels

---

# Best Practice

Keep labels simple and consistent.

It is better to use a few labels correctly than many labels incorrectly.

---

# Reminder

Labels help your team stay organized.

Use them every time you create an Issue.
