# Final Sprint Guide: Demo + Technical Defense + Handoff

## Purpose

The Final Sprint is about presenting a reliable project and proving team ownership.

You are not trying to add impressive new features. You are trying to make the project clear, stable, and defensible.

## Week 15: Rehearsal + Technical Defense

In class:

- run timed demo,
- test backup demo plan,
- freeze final scope,
- each student practices explaining one contribution,
- create final fix list.

## Week 16: Final Demo + Handoff

Final package must include:

- working demo or backup demo,
- final presentation,
- GitHub repo,
- updated README,
- setup guide,
- architecture/handoff doc,
- known bugs/limitations,
- individual contribution evidence,
- AI use / ownership statement.

## Technical Defense

Every student should be ready to answer:

- What did you personally contribute?
- Where is the evidence?
- What file/function/doc can you explain?
- What did AI help with?
- How did you test or verify your work?
- What is still weak or risky?

## Final Definition of Done

- [ ] Core flow can be demoed.
- [ ] Backup demo exists.
- [ ] Repo is organized.
- [ ] Docs are clear enough for handoff.
- [ ] Known bugs are documented.
- [ ] Every student has evidence.
- [ ] Team can explain AI-assisted work.

## Common Failure Modes

- Demo depends on one person’s laptop.
- Final app works once but cannot be repeated.
- Team hides bugs.
- Students cannot explain their own contributions.
- README does not explain setup.
