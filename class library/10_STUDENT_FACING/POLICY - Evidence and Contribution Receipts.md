# POLICY - Evidence and Contribution Receipts

## Purpose

This course grades visible professional work, not vague effort.

If it is not linked, it did not happen.

## What Counts as Evidence

Good evidence includes:

- PR link,
- Issue link,
- review comment link,
- CI/check run link,
- screenshot of working feature,
- demo video,
- test result,
- bug report,
- docs commit,
- deployment link.

Weak evidence includes:

- “I worked on it,”
- “We talked about it,”
- “Almost done,”
- “AI made it,”
- screenshot with no context,
- file only on someone’s laptop.

## Individual Contribution Receipts

Each student must comment on the Weekly Sprint Packet with **2–3 links**.

Copy/paste:

```md
## Contribution Receipts - Name

- Receipt 1:
- Receipt 2:
- Receipt 3:

### 1-sentence contribution summary

### AI Use Note
- AI tool used:
- What AI helped with:
- What I personally checked or changed:
- How I tested or verified it:
- One thing I still do not fully understand:
```

## Common Mistakes

- Linking only the team repo homepage.
- Linking a screenshot but not the Issue/PR.
- Claiming work without owner evidence.
- Submitting receipts after the deadline.
