# POLICY - Revision Windows

## Purpose

Revision windows are for improvement, not deadline erasure.

## Rules

- Missed work remains zero.
- Missing work must still be completed for course completion / bundle eligibility.
- Revisions are improvement-only.
- Revisions must link evidence.
- No vague resubmissions.

## Acceptable Revisions

Good:

- fixed missing evidence links,
- added screenshots/tests/PRs,
- clarified ownership,
- updated broken docs,
- closed feedback from instructor.

Weak:

- “We updated it” with no link.
- Reposting the same work.
- Adding a long AI-written explanation without evidence.
- Changing scope instead of fixing the assigned problem.

## Revision Submission Format

```md
# Revision Submission

## Original item
- Link:

## What was missing or Not Yet
-

## What improved
-

## Evidence links
- 
```
