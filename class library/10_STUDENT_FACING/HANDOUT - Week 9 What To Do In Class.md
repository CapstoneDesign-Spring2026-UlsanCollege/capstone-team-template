# Week 9 Handout: What To Do In Class

## Today’s Goal

Start Sprint 3 by proving what your project really is right now.

Not what you hope it is.  
Not what AI said it is.  
What actually runs, breaks, and can be explained.

## Step 1: Show One Working Flow

Pick the most important user flow.

Examples:

- user signs up and logs in,
- user creates a post/reservation/order,
- admin views submitted data,
- user completes the main project task.

Write:

- What flow did we try?
- Did it work?
- What broke?
- What evidence do we have?

## Step 2: Create the AI Code Ownership Audit

Create this file:

```txt
docs/AI_CODE_OWNERSHIP_AUDIT.md
```

Use the template from the class repo.

## Step 3: Make Bug Issues

Create Issues for:

- anything that blocks the final demo,
- anything that breaks the core flow,
- anything AI generated that no one understands,
- anything risky or fragile.

Use P0/P1/P2/P3 severity.

## Step 4: Update the Board

Your board should have:

- To Do
- Doing
- Done
- Blocked or Nice Later, if needed

Every active item needs:

- owner,
- Definition of Done,
- evidence plan.

## Step 5: Prepare the Weekly Progress Demo

Your team must show one visible improvement.

Use this format:

1. What changed?
2. Show the improvement.
3. Show the evidence.
4. Random member explains one code/doc/test area.
5. State one blocker or ask.

## Friday Submission

Submit one Weekly Sprint Packet Issue by Friday 23:59.

It must include:

- demo evidence,
- board snapshot,
- shipped items,
- bugs/risks,
- AI Use + Code Ownership Check,
- individual receipts in comments.
