# GUIDE - Wireframes

## Purpose
Wireframes turn your idea into something visible.

For this class, wireframes are **low-fidelity screen plans**. They show what the user sees and what the user can do.

## Main rule
**Plan the screen. Do not decorate the screen.**

Boxes, labels, arrows, and short notes are enough.

## What to make this week
For Week 4, your team should make **3 wireframes maximum**:

1. **Entry / home screen**
2. **Core task screen**
3. **Result / detail / confirmation / admin screen**

This forces you to focus on the MVP.

## What each wireframe should include
Every wireframe should show:
- screen title
- main content areas
- key buttons or actions
- labels for important sections
- short notes if needed

## What good wireframes do
Good wireframes make it easy to answer:
- What is the user trying to do here?
- What is the main action?
- What happens next?

## What to avoid
Do **not** waste time on:
- colors
- brand styling
- perfect spacing
- fancy icons
- detailed visual design
- ten extra screens you are not building now

## Example 3-screen flow
### Screen 1 - Home
- page title
- short description
- search/filter area
- button to continue

### Screen 2 - Main task
- main form or interaction area
- key data visible
- primary action button

### Screen 3 - Result
- confirmation / saved result / detail page
- next action
- error or fallback note if important

## Common mistakes
- too many screens
- no clear main action
- wireframes do not match the architecture
- wireframes show future features instead of MVP features
- making the screen pretty instead of useful

## Submission method
Put your wireframes in:
- `/docs/design-doc-v1.md`
- or linked image files inside `/docs/`
- and link them in the Weekly Sprint Packet

## Quick checklist
- [ ] only 3 screens max
- [ ] each screen has a clear purpose
- [ ] main action is obvious
- [ ] screens match the MVP
- [ ] screens match the architecture sketch
- [ ] labels are readable
- [ ] teammates can explain the flow in under a minute

## Final reminder
A wireframe is not the final design.

It is the fastest honest way to show what you plan to build.
