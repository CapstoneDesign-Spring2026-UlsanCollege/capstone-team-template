# CHECKLIST - Week 6 Repo Cleanup

## Purpose
Week 6 includes a hard stop for repo cleanup.

The goal is to remove confusion before Sprint 2 continues.
Your repo should make it easy to understand:
- what Sprint 2 is
- what this week’s packet is
- what feature matters most right now
- where the evidence is

---

## Naming Cleanup
### Required
- [ ] formal sprint document is named `SPRINT_2.md`
- [ ] weekly submission issue is titled `Weekly Sprint Packet — Week 6`
- [ ] visible “weekly sprint” wording has been removed where it causes confusion
- [ ] file and folder names clearly separate Sprint work from Week Packet work

### Check for confusion in
- repo files
- Issues
- board columns/cards
- docs folder
- README links

---

## Board Cleanup
- [ ] board has clear `To Do / Doing / Done` columns or equivalent
- [ ] active tasks have owners
- [ ] stale or confusing cards are cleaned up
- [ ] the midterm-critical feature is visible on the board
- [ ] board status matches reality

---

## Issue Cleanup
- [ ] important tasks have clear titles
- [ ] important tasks have one clear owner
- [ ] issue types make sense
- [ ] vague or duplicate issues are cleaned up
- [ ] midterm-critical feature has a clear Issue
- [ ] Definition of Done is written where needed

---

## PR Cleanup
- [ ] open PRs have meaningful titles
- [ ] linked Issue appears when appropriate
- [ ] `How tested` section is filled in
- [ ] PR descriptions are not left blank or half-finished

---

## Evidence Cleanup
- [ ] demo link exists and works
- [ ] backup demo plan exists
- [ ] board snapshot can be shown easily
- [ ] important work is linked in Issues or PRs
- [ ] proof is not scattered randomly without context

Rule:
**If it isn’t linked, it didn’t happen.**

---

## Midterm Readiness Cleanup
- [ ] team has identified one midterm-critical feature or flow
- [ ] current work supports the Midterm story
- [ ] roadmap is starting to make sense
- [ ] team roles are becoming clear

---

## Quick Room Check
Before class continues, your team should be able to show:
- [ ] `SPRINT_2.md`
- [ ] `Weekly Sprint Packet — Week 6`
- [ ] one main Issue for the Midterm feature
- [ ] one Definition of Done
- [ ] one testing/checking plan
- [ ] one trail of evidence

---

## Common Problems to Fix Immediately
Fix these now if you see them:
- confusing file names
- weekly packet called a sprint
- Issues with no owner
- PRs with empty testing notes
- fake “Done” cards
- board and repo telling different stories
- too many tasks with no priority

---

## Final Reminder
This cleanup is not busywork.

It is part of making your project:
- easier to manage
- easier to grade
- easier to present
- easier to trust
